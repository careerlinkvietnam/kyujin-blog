# リライトマニュアル

**重要**: リライト作業を行う前に、必ずこのマニュアルを読むこと。

> **基本ルールは `00_必読ルール.md` を参照。**

---

## 基本原則

1. **リライトは慎重に行う** - 既存記事には価値がある。安易に書き換えない。
2. **元記事を尊重する** - 元記事の良い部分は必ず残す。
3. **追加・改善のみ行う** - 削除や全面書き換えは原則禁止。
4. **量より質** - 情報量を増やすことより、情報の質を高める。
5. **必ずバックアップを取る** - いつでも元の記事に戻せるようにする。

---

## バージョン管理（必須）

**リライト前に必ず元記事をバックアップし、いつでも元に戻せる状態にすること。**

### バックアップ手順

#### 1. 元記事をWordPress APIから取得して保存

```bash
curl -s "https://kyujin.careerlink.asia/blog/wp-json/wp/v2/posts/[記事ID]" \
  -u "careerlinkasia:N2Zz bzSn AWve Pa83 Ap2S 6Mlw" \
  > backups/articles/[slug]_v[バージョン]_[日付].json
```

#### 2. バージョン履歴に記録

`backups/バージョン履歴.md` に以下を記録：
- 記事ID
- バックアップファイル名
- バックアップ日時

#### 3. バックアップ確認

保存したファイルを開いて、以下を確認：
- `title.rendered` が正しいか
- `content.rendered` が正しいか

### 復元手順（ロールバック）

リライト後に問題が発覚した場合、以下の手順で元に戻す：

```javascript
// Node.js で実行
const fs = require('fs');
const d = JSON.parse(fs.readFileSync('backups/articles/[バックアップファイル].json', 'utf-8'));
const restore = {
    title: d.title.rendered,
    content: d.content.rendered,
    excerpt: d.excerpt.rendered
};
fs.writeFileSync('restore_[slug].json', JSON.stringify(restore), 'utf-8');
```

```bash
curl -s -X PUT "https://kyujin.careerlink.asia/blog/wp-json/wp/v2/posts/[記事ID]" \
  -u "careerlinkasia:N2Zz bzSn AWve Pa83 Ap2S 6Mlw" \
  -H "Content-Type: application/json" \
  -d @restore_[slug].json
```

## リライト前チェックリスト（必須）

### Step 0: バックアップを取る

- [ ] WordPress APIから元記事を取得
- [ ] `backups/articles/` に保存
- [ ] `backups/バージョン履歴.md` に記録
- [ ] バックアップファイルの内容を確認

**バックアップを取らずにリライトを開始してはいけない。**

### Step 1: 元記事の詳細分析

リライト前に以下を必ず確認し、記録すること：

- [ ] 元記事の文字数
- [ ] 元記事の見出し構成
- [ ] 元記事で紹介している企業・サービスの数
- [ ] 元記事の具体的な情報（連絡先、住所、料金など）
- [ ] 元記事の独自情報（取材内容、体験談など）
- [ ] 元記事の内部リンク・外部リンク
- [ ] 元記事のCTA

### Step 2: リライトが本当に必要か判断

| 状況 | 対応 |
|-----|------|
| 情報が古い（年号、料金、サービス内容） | 該当部分のみ更新 |
| CTAがない・弱い | CTAのみ追加 |
| 内部リンクがない | 内部リンクのみ追加 |
| 順位が低い | まず原因分析。構成に問題がなければ様子見 |
| 内容が薄い | 情報を追加（削除はしない） |

**全面リライトが必要なケース（稀）**:
- 記事の方向性自体が間違っている
- 検索意図と完全にずれている
- 情報が全体的に古く、修正箇所が8割を超える

### Step 3: リライト計画書の作成

全面リライトを行う場合、以下の計画書を作成すること：

- 記事ID:
- 現在の文字数:
- 現在の企業/サービス紹介数:

### 元記事の強み（必ず残す部分）
1.
2.

### 元記事の弱み（改善する部分）
1.
2.

### リライト方針
- [ ] 部分修正（推奨）
- [ ] 全面リライト（要承認）

## リライト実行時のルール

### やってはいけないこと

1. **元記事を読まずにリライトする**
2. **元記事の具体的情報を削除する**
3. **企業・サービスの紹介数を減らす**
4. **汎用的な説明文で置き換える**
5. **調査せずに情報を追加する**
6. **年号だけ更新して内容は同じにする**

### やるべきこと

1. **元記事を完全に読み込む**
2. **元記事の良い部分をリストアップする**
3. **追加情報は調査して正確性を確認する**
4. **変更前後で文字数・情報量を比較する**
5. **リライト後、元記事と並べて品質比較する**

## リライト後チェックリスト（必須）

投稿前に以下を確認：

- [ ] 文字数: 元記事 ≦ リライト後
- [ ] 企業/サービス数: 元記事 ≦ リライト後
- [ ] 具体的情報: 元記事の情報がすべて含まれている
- [ ] 独自情報: 元記事の独自情報が残っている
- [ ] 品質: 汎用的な説明だけになっていない
- [ ] CTA: 適切に配置されている
- [ ] 内部リンク: 関連記事へのリンクがある

**上記すべてにチェックが入らない場合、リライトを投稿しない。**

## リライト品質比較（最重要）

### 絶対ルール

**リライト後の記事が、リライト前の記事より品質が高い場合のみ更新する。**

品質が同等または低下した場合、更新しない。

### 品質比較の手順

#### Step 1: 定量比較

| 項目 | 元記事 | リライト後 | 判定 |
|------|-------|----------|------|
| 文字数 | | | 元記事 ≦ リライト後 → OK |
| 見出し数 | | | 元記事 ≦ リライト後 → OK |
| 紹介企業/サービス数 | | | 元記事 ≦ リライト後 → OK |
| 具体的情報数（住所、電話番号、料金等） | | | 元記事 ≦ リライト後 → OK |
| 内部リンク数 | | | 元記事 ≦ リライト後 → OK |

**1項目でも「元記事 > リライト後」があれば、原則として更新しない。**

#### Step 2: 定性比較

| 観点 | 質問 | 判定基準 |
|------|------|---------|
| 具体性 | 汎用的な説明が増えていないか？ | 具体的情報が減っていれば NG |
| 独自性 | 元記事の独自情報が消えていないか？ | 独自情報が消えていれば NG |
| 実用性 | 読者が行動しやすくなったか？ | 行動しにくくなっていれば NG |
| 読みやすさ | 構成・文章は改善されたか？ | 悪化していれば NG |
| 正確性 | 情報は正確か？ | 不正確な情報があれば NG |

## CTA追加ルール

観光・生活記事にCTAを追加する際は、以下のルールに従うこと。

### リンク先の選定

| 記事の対象国 | CTAのタイトル | リンク先 |
|------------|--------------|---------|
| タイ | 「タイで働くことに興味はありませんか？」 | https://kyujin.careerlink.asia/thailand/job/list |
| ベトナム | 「ベトナムで働くことに興味はありませんか？」 | https://kyujin.careerlink.asia/vietnam/job/list |
| インドネシア | 「インドネシアで働くことに興味はありませんか？」 | https://kyujin.careerlink.asia/indonesia/job/list |
| **その他・複数国** | 「東南アジアで働くことに興味はありませんか？」 | https://kyujin.careerlink.asia/job/list |

### CTAテンプレート

```html
<div style="background-color: #f0f8ff; border: 2px solid #1e90ff; border-radius: 10px; padding: 20px; margin: 30px 0;">
<h3 style="color: #1e90ff; margin-top: 0;">[タイトル]</h3>
<p>[説明文]</p>
<p><strong><a href="[リンク先]" style="color: #1e90ff;">▶ [国名]の求人一覧を見る</a></strong></p>
<p><strong><a href="https://kyujin.careerlink.asia/register" style="color: #1e90ff;">▶ 無料会員登録はこちら</a></strong></p>
</div>
```

### CTA追加時の注意

1. **元記事の内容は変更しない** - CTAは記事の最後に追加するのみ
2. **バックアップを取る** - CTA追加前に必ずバックアップを保存
3. **国に応じたリンク先を選択** - 上記表に従う

## 過去の失敗事例

### 2026-01-17: タイ・ベトナム人材紹介会社

**何が起きたか**:
- 元記事を十分に分析せずにリライト
- 元記事より情報量が減少
- 汎用的な説明文になり、具体性が失われた
- 結果: 質が著しく低下し、ロールバック

**教訓**:
- 元記事の詳細分析は必須
- 「新しい=良い」ではない
- 量を増やすことより、元の質を維持することが重要
