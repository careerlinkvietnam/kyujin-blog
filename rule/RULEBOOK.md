## 1. サイトの基本思想

### 1.1 B2B専用である

このサイト（kyujin.careerlink.asia/blog 配下）のB2Bコンテンツは、**企業の人事担当者・海外拠点責任者のみ**を対象とする。

- 求職者向け要素は一切含めない
- 「転職したい人」が読んでも得るものがない設計にする
- 問い合わせ導線は企業向けのみ

### 1.2 「判断LP」という考え方

このサイトのLPは「CVを最大化する装置」ではない。

**LPの役割**:
- 企業が採用すべきかどうかを判断できる状態にする
- 判断が終わった人が、迷わず問い合わせできる状態にする

**LPがやらないこと**:
- 問い合わせを増やすための煽り
- 「今すぐ」「簡単」などの文言追加
- バナー広告的なCTA配置

### 1.3 売り込まない

- 我々の役割は「売る」ことではない
- 企業が正しく判断できる状態を作ることが役割
- 「検討します」で引き延ばさない。判断を出す。

## 2. LP構成の設計思想

### 2.1 総論LP vs 国別LP

| LP種別 | 対象 | URL | 役割 |
|--------|------|-----|------|
| 総論LP | タイ・ベトナム両方を検討中 | /blog/b2b-recruitment-consulting/ | 全体像の提示、比較軸の提供 |
| 国別LP | 特定国に絞って検討中 | /blog/vietnam-recruitment-consulting/ 等 | 国固有の論点に特化 |

**設計原則**:
- 総論LPは「どちらの国にすべきか」の判断材料を提供
- 国別LPは「その国で採用できるか」の判断材料を提供
- 両者は独立して機能する（相互参照不要）

### 2.2 国別LPの差別化

ベトナムLPの例（Phase B-1で作成）:
- ベトナム固有の論点を5つに整理
  1. 日本語要件（N2人材の争奪戦）
  2. ホーチミン・ハノイの給与相場差
  3. 若手人材の離職リスク
  4. 管理職の現地採用可否
  5. 求人広告 vs 人材紹介の使い分け

- タイの論点とは異なる（タイは製造業・EEC・BOI等が論点になる）

## 3. 内部リンク設計

### 3.1 Type A/B/C 分類（Phase 12で確定）

既存記事からLPへのリンクは、記事の性格によって3分類する。

| Type | 対象記事 | リンク要否 | 理由 |
|------|----------|------------|------|
| Type A | 企業人事が判断で止まる記事 | 必須 | 採用ガイド、労働法、給与制度など |
| Type B | 制度・市場解説系 | 条件付き | 祝日、赴任準備、福利厚生など |
| Type C | 求職者向け・一般情報 | 不要 | 転職体験談、観光情報など |

### 3.2 リンクパターン（Phase 12で確定）

**パターンA（採用ガイド系）**:
> 本記事で採用の流れや制度を把握した上で、「この条件で本当に採用できるのか」「給与設定が市場に合っているか」といった判断に迷う場合は、採用難易度・給与レンジを事実ベースで整理する無料相談をご活用ください。

**パターンB（労働法・制度系）**:
> 制度や規定を理解した上で、「自社の条件で採用が現実的か」「どのような人材紹介会社に依頼すべきか」といった判断に迷う場合は、採用難易度・給与レンジを事実ベースで整理する無料相談をご活用ください。

**パターンD（Type B用・市場解説系）**:
> なお、採用の判断に迷う場合は、採用難易度・給与レンジを事実ベースで整理する無料相談も行っています。

### 3.3 リンク挿入位置

- まとめセクションの直前
- または記事末尾
- 本文中への散発的挿入は禁止

## 4. CTA設計思想

### 4.1 基本原則

CTAは「増やす」のではなく「見落とさせない」ことが目的。

**確定ルール**:
- CTAは1箇所に集約する
- 配置位置は「判断完了地点」直後
  - 費用条件 → 秘密厳守 → 返信目安 → **CTA**
- ボタンUIは最小限（青色・角丸・中央配置）
- 影・アニメーション・アイコン追加は禁止

### 4.2 禁止事項

- ヒーロー直下へのCTA配置
- ページ中間への複数CTA
- 「今すぐ」「簡単」「無料」を強調する文言
- モーダル表示・フォーム埋め込み
- バナー広告的なデザイン

### 4.3 CTA文言（固定）

```
問い合わせフォームから無料相談する
```

この文言は変更しない。URLも固定:
```
https://kyujin.careerlink.asia/お問い合わせ
```

## 5. 公開・検証の考え方

### 5.1 公開前チェック

- [ ] CTAは1箇所のみか
- [ ] CTA位置は判断完了地点直後か
- [ ] フォームURLは正しいか
- [ ] モバイル表示で押しづらくないか
- [ ] タイ・ベトナムの混同がないか（国別LPの場合）

### 5.2 公開後チェック

- [ ] ページが正常に表示される
- [ ] CTAリンクが正常に動作する
- [ ] Search Console で URL検査実行

### 5.3 効果測定の考え方

CVRを追わない。以下を確認する:
- 問い合わせの質（A/B/C分類比率）
- 初回ヒアリングで必須5項目が揃う率
- 「条件調整」で終わる比率

## 7. 判断の根拠となる原則

### 7.1 迷ったときの判断基準

1. **それは企業の判断を助けるか？** → 助けないなら削除
2. **それは売り込みに見えるか？** → 見えるなら削除
3. **それは求職者向けか？** → そうなら対象外

### 7.2 やってはいけないこと

- 「すべての問い合わせを取りに行く」設計
- 複数CTAで選択肢を増やす
- 煽り文言で行動を促す
- 判断を先送りにする対応
- 全員に同じ温度で対応する

### 7.3 優先順位

1. 企業が正しく判断できる
2. 判断が終わった人が迷わず問い合わせできる
3. 問い合わせ後に適切な対応ができる

CVR・問い合わせ数は優先順位に含まれない。
