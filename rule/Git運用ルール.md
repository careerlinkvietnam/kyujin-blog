# Git運用ルール

**最終更新**: 2026-01-24

---

## 基本原則

すべての記事リライト・新版作成はGitで管理する。
CMS（WordPress）は最終反映のみ。**Gitが品質の正本**である。

---

## ブランチ戦略

```
main                         ← 現行公開版のみ
├── article/{記事ID}-v{年}   ← 作業用ブランチ
└── review/{記事ID}          ← 品質確認用（任意）
```

### ブランチ命名規則

| ブランチ種別 | 命名パターン | 例 |
|-------------|-------------|-----|
| 本番 | `main` | - |
| 記事作業 | `article/{記事ID}-v{年}` | `article/6992-v2026` |
| レビュー | `review/{記事ID}` | `review/6992` |
| 新規記事 | `article/{記事ID}-new` | `article/8563-new` |

---

## 記事ディレクトリ構造

```
articles/
├── {記事ID}/
│   ├── v{年}_旧版.md        ← 現行公開内容のスナップショット
│   ├── v{年}_draft.md       ← Claudeが生成したドラフト
│   └── compare.md           ← 品質比較表（必須）
```

### 必須ファイル

| ファイル | 必須 | 説明 |
|---------|------|------|
| `v{年}_旧版.md` | リライト時 | 現行公開版の構造記録 |
| `v{年}_draft.md` | 常に | 新版ドラフト |
| `compare.md` | 常に | 品質比較表（これがないとマージ禁止） |

---

## 品質比較ルール

### compare.md の必須評価項目

1. **情報量**: 削減されていないか
2. **具体性**: 抽象化されていないか
3. **B2B意思決定視点**: 企業人事向けの視点があるか
4. **一次情報・根拠**: 参照情報が明示されているか
5. **読後の行動**: 相談したくなる設計になっているか

### 判定基準

| 結果 | 対応 |
|-----|------|
| 全項目「強化」or「適合」 | ✅ 合格 → mainにマージ可 |
| 1項目でも「劣化」 | ❌ 不合格 → Draft修正 or 破棄 |

---

## 公開判定フロー

```
1. article/{記事ID}-v{年} ブランチで作業
   ↓
2. compare.md を作成・評価
   ↓
3-A. 合格 → main にマージ → WordPress へ反映
3-B. 不合格 → main に触れない → 旧記事継続公開
```

## コミットメッセージ規則

```
[記事ID] アクション: 概要

例:
[6992] draft: v2026ドラフト作成
[6992] compare: 品質比較表作成
[6992] fix: FAQ追加
[6992] merge: v2026をmainにマージ
```

### アクション一覧

| アクション | 用途 |
|-----------|------|
| `draft` | ドラフト作成・更新 |
| `compare` | 比較表作成・更新 |
| `fix` | ドラフト修正 |
| `merge` | mainへのマージ |
| `snapshot` | 旧版スナップショット作成 |

---

## WordPress反映手順

1. mainブランチに該当記事のcompare.mdが存在することを確認
2. compare.mdの判定が「合格」であることを確認
3. draft HTMLファイルをWordPressに反映
4. 反映完了後、コミットに反映日時を記録
