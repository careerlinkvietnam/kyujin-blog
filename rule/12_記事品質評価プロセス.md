# 記事品質評価プロセス

**作成日**: 2026-01-21
**目的**: 記事作成後の品質評価を標準化し、品質基準を確実に満たすことを保証する

---

## 1. 品質評価の実施タイミング

**必須**: 記事作成完了直後、次の記事に移る前に必ず実施する

```
記事作成 → 品質評価 → [基準未達なら修正] → 基準達成 → 次の記事へ
```

---

## 2. 品質評価チェックリスト

### A. 一次情報チェック

| チェック項目 | 確認内容 | 必須/推奨 |
|-------------|---------|----------|
| □ 法律・規制 | 政府公式サイト（労働省、移民局等）を参照したか | 必須 |
| □ 統計データ | 公式統計局のデータを使用したか | 必須 |
| □ 最新性 | データの年度は2025-2026年か | 必須 |
| □ 現地語情報 | 現地語（タイ語/ベトナム語）の一次情報を確認したか | 推奨 |
| □ 出典明記 | 重要な数字・事実に出典を明記したか | 必須 |
| □ 一次情報リンク | 政府サイトへの直接リンクがあるか | 推奨 |

**使用OK**: 政府機関、JETRO、大使館、公式統計局
**使用NG**: コンサル会社、法律事務所コラム、ニュースサイトの解説

### B. 文字数・構成チェック

| チェック項目 | 基準 | 確認方法 |
|-------------|------|---------|
| □ 文字数 | 20,000文字以上 | ファイル行数から概算（1行約30文字×行数）または文字数カウント |
| □ 構成 | 論理的な見出し構成（H2→H3の階層） | 目視確認 |
| □ 手動目次 | 含まれていないこと | 目視確認 |
| □ 表の活用 | 重要情報は表形式で整理 | 目視確認 |

### C. 内部リンクチェック

| チェック項目 | 確認内容 |
|-------------|---------|
| □ 関連記事リンク | 関連する既存記事へのリンクが含まれているか |
| □ 求人ページリンク | 国別求人ページへのリンクがあるか |

**内部リンク挿入箇所**:
- 労働法に言及 → 労働法ガイドへリンク
- ビザ・WPに言及 → ビザガイドへリンク
- まとめセクション → 関連記事一覧を追加

### D. CTAチェック

| チェック項目 | 基準 |
|-------------|------|
| □ CTAリンク先 | ターゲットに合ったリンク先か（求職者→/register、企業→/register?type=employer） |
| □ CTA数 | 最大2個以内 |
| □ 連続CTA | 2つのCTAが連続していないか |

### E. コンプライアンスチェック

| チェック項目 | 確認内容 |
|-------------|---------|
| □ 禁止ワード | 「法人設立」「投資アドバイス」を使用していないか |
| □ 政府批判 | 政府・政策への批判的記述がないか |
| □ 日系企業批判 | クライアント企業への批判がないか |
| □ URLスラッグ | 日本語をURLに使用していないか |

## 3. 品質評価の実施手順

### Step 1: 記事を読み込んで基本チェック

```
1. Readツールで作成した記事を読み込む
2. 文字数を確認（目安：500行以上で約15,000文字、700行以上で約20,000文字）
3. 構成（見出し階層）を確認
```

### Step 2: チェックリストに沿って評価

チェックリストA〜Eの各項目を確認し、以下の形式で記録:

```markdown
## 品質評価結果

### [記事タイトル]
- **文字数**: 約XX,XXX文字 ✅/❌（基準20,000文字以上）
- **一次情報**: [参照した情報源を記載] ✅/❌
- **最新性**: 20XX年データ ✅/❌
- **内部リンク**: [有無] ✅/❌
- **CTA**: [正しい/修正要] ✅/❌
- **禁止ワード**: なし/あり ✅/❌

### 改善が必要な点
1. [改善点1]
2. [改善点2]
```

### Step 3: 基準未達の場合は修正

**必須修正**（これがないと不合格）:
- 文字数が20,000文字未満
- 一次情報の参照がない
- CTAリンク先が間違っている
- 禁止ワードが含まれている
- 2024年以前の古いデータを使用

**推奨修正**（品質向上のため）:
- 内部リンクがない
- 政府サイトへの直接リンクがない
- 表が少ない

### Step 4: 修正完了後、次の記事へ

```
修正完了 → todoを「completed」に更新 → 次の記事作成開始
```

## 6. よくある品質問題と対処法

| 問題 | 対処法 |
|------|-------|
| 文字数不足 | 業務内容の詳細化、業種別の解説追加、FAQ追加 |
| 一次情報なし | 法律名・政令番号を追記、政府サイトURLを追加 |
| 内部リンクなし | 関連記事セクションを追加、本文中に参照リンク挿入 |
| データが古い | 最新の年次に更新（2025-2026年） |
| CTAが多い | 2個以内に削減 |
