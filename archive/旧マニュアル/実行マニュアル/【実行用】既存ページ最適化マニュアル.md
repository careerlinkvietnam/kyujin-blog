# 既存ページ最適化マニュアル Ver 1.0
## Claude Code による自律実行仕様

---

## 1. 目的

新規記事を作る前に、**既存の高トラフィックページをCV（会員登録・応募）に繋がる状態に改善する**。

「トラフィックはあるが転職に無関係な記事」を「転職CVへの入口」に変換する。

---

## 2. 実行条件

### 前提条件
- 「サイト全体診断マニュアル」が実行済みであること
- 診断結果に「高トラフィック非CTA記事」が特定されていること

### 実行トリガー
- 診断結果で優先度S/Aタスクとして生成された場合
- ユーザーからの指示時

---

## 3. 対象ページの選定

### 3.1 優先度決定マトリクス

| GSCクリック数/月 | 転職関連度 | アクション | 優先度 |
|-----------------|-----------|-----------|-------|
| 500以上 | 低（観光・生活系） | CTA注入 + 内部リンク追加 | S |
| 500以上 | 高（転職系） | メタタグ最適化 + リライト | A |
| 100-500 | 低 | CTA注入のみ | B |
| 100-500 | 高 | メタタグ最適化のみ | B |
| 100未満 | 任意 | 対象外（効率悪い） | - |

### 3.2 GSCデータからの抽出

```
対象ページリスト = GSCページデータ
    .filter(クリック数 >= 100)
    .filter(URL.includes("/blog/"))
    .sort(クリック数, 降順)
    .take(50)
```

## 4. CTA注入

### 4.1 CTAテンプレート

#### パターンA: 記事中盤挿入用（H2タグの後）

```html
<div style="background: #f8f9fa; border-left: 4px solid #007bff; padding: 20px; margin: 30px 0;">

  <p style="margin-bottom: 15px;">現地の生活情報を調べているあなたへ。「働く」という選択肢も検討してみませんか？</p>
  <p><a href="${JOB_LIST_URL}" style="color: #007bff; text-decoration: underline;">→ ${COUNTRY_JP}の最新求人を見る</a></p>
</div>
```

```html
<div style="background: #e8f4fd; padding: 25px; margin-top: 40px; border-radius: 8px;">
```

#### パターンC: 関連記事リンク用

```html

  </ul>
</div>
```

### 4.2 国・地域判定ロジック

記事URL・タイトル・本文から判定：

```
IF 記事に含まれるキーワード:
    "ハノイ" OR "ホーチミン" OR "ベトナム" OR "vietnam"
    → COUNTRY = "Vietnam"
    → COUNTRY_JP = "ベトナム"
    → JOB_LIST_URL = "https://kyujin.careerlink.asia/vietnam/job/list"

    "バンコク" OR "タイ" OR "thailand"
    → COUNTRY = "Thailand"
    → COUNTRY_JP = "タイ"
    → JOB_LIST_URL = "https://kyujin.careerlink.asia/thailand/job/list"

    "シンガポール" OR "singapore"
    → COUNTRY_JP = "シンガポール"
    → JOB_LIST_URL = "https://kyujin.careerlink.asia/singapore/job/list"

    "マレーシア" OR "クアラルンプール" OR "malaysia"
    → COUNTRY_JP = "マレーシア"
    → JOB_LIST_URL = "https://kyujin.careerlink.asia/malaysia/job/list"

    "インドネシア" OR "ジャカルタ" OR "indonesia"
    → COUNTRY_JP = "インドネシア"
    → JOB_LIST_URL = "https://kyujin.careerlink.asia/indonesia/job/list"

    "カンボジア" OR "プノンペン" OR "cambodia"
    → COUNTRY_JP = "カンボジア"
    → JOB_LIST_URL = "https://kyujin.careerlink.asia/cambodia/job/list"

ELSE:
    → COUNTRY_JP = "東南アジア"
    → JOB_LIST_URL = "https://kyujin.careerlink.asia/job/list"
```

### 4.3 CTA挿入位置の決定

```
1. 記事本文をHTMLとしてパース
2. H2タグを検出
3. 2番目のH2タグの直前にパターンAを挿入
4. 記事末尾（最後のPタグの後）にパターンBを挿入
5. 既存の「関連記事」セクションがない場合、パターンCを挿入
```

### 4.4 挿入ルール（厳守）

- **同一CTAの重複禁止**: 既にCTAが存在する場合は追加しない
- **最大挿入数**: 1記事あたりCTA最大2箇所
- **jobseeker/register リンクの確認**: 正しいURLを使用（/register ではない）

## 5. 内部リンク注入

### 5.1 リンク候補キーワードと挿入先

| 検出キーワード | 挿入リンク | アンカーテキスト |
|--------------|-----------|----------------|
| "ベトナムで働" | /vietnam/job/list | ベトナムの求人を見る |
| "タイで就職" | /thailand/job/list | タイの求人を見る |
| "海外転職" | /blog/overseas-job-40s-guide/ | 海外転職ガイド |

| "40代" / "ミドル" | /blog/overseas-job-40s-guide/ | 40代からの海外転職 |

| "女性" | /blog/overseas-job-women-guide/ | 女性の海外転職ガイド |

### 5.2 注入ルール

```
1. 記事本文から上記キーワードを検出
2. 該当キーワードが「既にリンク化されていない」ことを確認
3. 最初に出現した箇所のみリンク化（同一キーワードの重複リンク禁止）
4. 1記事あたり最大5件の内部リンク追加
```

### 5.3 リンク挿入コード例

```html
<!-- Before -->
<p>ベトナムで働くことを検討している方も多いでしょう。</p>

<!-- After -->
<p><a href="https://kyujin.careerlink.asia/vietnam/job/list">ベトナムで働く</a>ことを検討している方も多いでしょう。</p>
```

## 6. 実行フロー

### Step 1: 対象ページリストの取得

```
1. 診断レポートから「高トラフィック非CTA記事」リストを取得
2. または、GSCページデータから直接抽出
3. 優先度順にソート
```

### Step 2: WordPress記事の取得

```
1. WordPress REST API で記事本文を取得
   GET /wp-json/wp/v2/posts?slug={slug}

2. 認証情報:
   - ユーザー名: careerlinkasia
   - パスワード: プロジェクト進捗履歴.md 参照
```

### Step 3: 変更内容の生成

```
1. 国・地域を判定
2. CTAテンプレートを選択・変数展開
3. 挿入位置を決定
4. 内部リンク候補を抽出
5. 変更後HTMLを生成
```

### Step 4: プレビュー出力（必須）

```
変更内容を以下のファイルに出力：
out/page_optimization_preview_YYYY-MM-DD.md

内容:
- 対象URL
- 変更前後の差分
- 追加されるCTA
- 追加される内部リンク
```

### Step 5: 人間の承認

```
プレビューファイルをユーザーに提示
「この変更を適用してよいですか？」と確認
```

### Step 6: WordPress更新実行

```
承認後のみ実行:
PUT /wp-json/wp/v2/posts/{id}
{
  "content": "{updated_content}"
}
```

### Step 7: 結果記録

```
プロジェクト進捗履歴.md に以下を追記:
- 更新日時
- 対象ページURL
- 追加したCTA数
- 追加した内部リンク数
```

## 9. 安全ルール（厳守）

1. **プレビュー必須**
   - 本番更新前に必ずプレビューファイルを出力
   - 人間の承認なしに更新しない

2. **バックアップ**
   - 更新前の記事本文を `backup/` フォルダに保存

3. **1日の更新上限**
   - 最大10記事/日
   - API負荷とレビュー負荷を考慮

4. **リンクの確認**
   - 挿入するすべてのURLが有効であることを事前確認
   - 特に `/jobseeker/register` の正確なパスを使用

5. **ロールバック手順**
   - 問題発生時は `backup/` から復元可能な状態を維持
