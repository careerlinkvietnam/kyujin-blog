# Git運用ルール

**最終更新**: 2026-01-24

---

## 基本原則

すべての記事リライト・新版作成はGitで管理する。
CMS（WordPress）は最終反映のみ。**Gitが品質の正本**である。

---

## 品質管理の最重要ルール

### Claudeの役割制限

1. **Claudeは「合否判定」を行ってはいけない**
2. 公開可否は**必ず人間が決定する**
3. compare.mdは**判断材料**であり、**結論ではない**

### 公開フロー

```
Claude: compare.md作成（判断材料の提示のみ）
  ↓
人間: レビュー・判定
  ↓
人間が合格判定 → WordPress反映
人間が不合格判定 → 旧版維持、ドラフト破棄または修正
```

---

## ブランチ戦略

```
main                         ← 現行公開版のみ
├── article/{記事ID}-v{年}   ← 作業用ブランチ
└── review/{記事ID}          ← 品質確認用（任意）
```

### ブランチ命名規則

| ブランチ種別 | 命名パターン | 例 |
|-------------|-------------|-----|
| 本番 | `main` | - |
| 記事作業 | `article/{記事ID}-v{年}` | `article/6992-v2026` |
| レビュー | `review/{記事ID}` | `review/6992` |
| 新規記事 | `article/{記事ID}-new` | `article/8563-new` |

---

## 記事ディレクトリ構造

```
articles/
├── {記事ID}/
│   ├── v{年}_正本.md         ← 現行公開版（変更禁止）
│   ├── v{年}_draft.md        ← ドラフト（参考資料）
│   ├── v旧版_content.html    ← 実際のHTMLコンテンツ
│   └── compare.md            ← 品質比較表（判断材料）
```

### 必須ファイル

| ファイル | 必須 | 説明 |
|---------|------|------|
| `v{年}_正本.md` | 常に | 現行公開版のメタ情報 |
| `v旧版_content.html` | 常に | 実際のHTMLコンテンツ |
| `compare.md` | 常に | 品質比較表（人間判断用） |

---

## 品質比較ルール

### compare.md の必須評価項目

1. **情報量**: 削減されていないか
2. **具体性**: 抽象化されていないか
3. **B2B意思決定視点**: 企業人事向けの視点があるか
4. **一次情報・根拠**: 参照情報が明示されているか
5. **読後の行動**: 相談したくなる設計になっているか

### 判定プロセス

| ステップ | 実行者 | 内容 |
|---------|--------|------|
| 1 | Claude | 比較項目を分析し、compare.mdを作成 |
| 2 | Claude | 「強化」「維持」「劣化」を項目ごとに記載 |
| 3 | **人間** | レビューし、最終判定を下す |
| 4 | **人間** | 合格なら公開指示、不合格なら却下 |
