## 1. 最終目的と手段

### 最終目的

**企業の人事担当者が「この会社に相談しよう」と判断できる状態を作ること。**

| やること | やらないこと |
|----------|--------------|
| 採用の判断材料を提供する | 問い合わせ数を最大化する |
| 正確な市場情報を伝える | 煽って行動させる |
| 信頼される情報サイトを作る | 売り込む |

### 顧客の優先順位

| 優先度 | 顧客 |
|-------|------|
| **1位** | **日系企業の人事担当者** |
| 2位 | 日系企業の駐在員 |
| 3位 | 日本人求職者 |
| 4位 | 現地滞在者（①〜③以外） |
| 5位 | 観光客 |

**B2B（1位・2位）が最優先。求職者向けコンテンツは人材プールを構築する手段。**

### 手段：東南アジア最高の情報サイトを作る

最終目的を達成するには、**「このサイトを見れば間違いない」と言われる情報サイトになる必要がある。**

最高のコンテンツがあるからこそ：
- 企業の人事担当者が信頼して相談する
- 求職者が集まり、人材プールが構築される
- Googleで上位表示され、認知が広がる

### 対象範囲（転職・就職だけではない）

| 分野 | 例 |
|------|-----|
| 転職・就職 | 求人ガイド、人材紹介会社比較、年収相場 |
| 政治・行政 | 政府動向、政策変更、選挙 |
| 経済 | GDP、投資環境、産業動向 |
| 法律 | 労働法、外資規制、ビザ・労働許可 |
| 経理・会計 | 税制、社会保険、給与計算 |
| 社会・文化 | 祝日、マナー、宗教、国民性 |
| 地方 | 工業団地、地方都市、不動産 |
| 生活 | 生活費、住居、医療、教育 |
| 観光 | 都市ガイド、ホテル、移動手段 |

**すべての分野で、日本語で読める最高の情報を提供する。**

### 競合に勝つ唯一の方法

日本語の情報だけを参照していては、絶対に日本一のサイトにはなれない。

- 一次情報（政府・公的機関）を参照する
- 現地語（タイ語・ベトナム語）の情報を参照する
- 競合サイトより網羅的・正確・最新の情報を提供する

## 3. 絶対禁止事項

| # | 禁止事項 | 理由 |
|---|---------|------|
| 1 | 「法人設立」サービスを提案する | 人材紹介会社であり、法人設立サービスは提供していない |
| 2 | 「投資アドバイス」を提供する | 投資コンサルタントではない |
| 3 | 政府・政策を批判する | 現地での事業継続リスク |
| 4 | 架空のデータを捏造する | 「n=500」「CareerLink Asia調べ」など禁止 |
| 5 | 日系企業を批判する | クライアント企業のため |
| 6 | コンサル会社を出典にする | 一次情報のみ使用可 |
| 7 | 手動目次を追加する | WordPressプラグインと重複 |
| 8 | 日本語をURLスラッグにする | SEOに悪影響 |
| 9 | 元記事を読まずにリライトする | 品質低下の原因 |
| 10 | バックアップなしでリライトする | ロールバック不可能 |
| 11 | ファイル更新後にコミットしない | 変更履歴が追跡不可能 |
| 12 | 目次プラグイン直後に`<div>`を配置する | 目次のスタイルと干渉してレイアウトが崩れる |

### 目次プラグインとの干渉について

WordPressの目次プラグインは、最初の`<h2>`タグの直前に目次を自動挿入する。目次の直後（＝導入文の最後〜最初のh2の間）に`<div>`要素を配置すると、目次のスタイルと干渉してレイアウトが崩れる。

**対策:**
- 導入文直後のボックス要素は`<div>`で囲まず、`<p>`などの要素を直接使用する
- または、ボックス要素を最初の`<h2>`の後に配置する

### 「この記事でわかること」セクションの正しい書き方（重要）

**禁止事項:**
- `<div>`で囲む → 目次と干渉してレイアウト崩れ
- `<h3>`や`<h4>`を使う → 目次に表示されてしまう
- 最初の`<h2>`より前に配置する → 目次より上に表示されてしまう

**正しい形式:**
```html
<p>導入文...</p>

<h2>〇〇の概要</h2>

<p><strong style="color: #2e7d32; font-size: 1.1em;">この記事でわかること</strong></p>
<ul style="background: linear-gradient(135deg, #e8f5e9 0%, #c8e6c9 100%); border-radius: 12px; padding: 25px 25px 25px 45px; margin: 0 0 30px 0; color: #1b5e20;">
<li>ポイント1</li>
<li>ポイント2</li>
</ul>

<h3>詳細セクション</h3>
```

**ポイント:**
1. 「この記事でわかること」は最初の`<h2>`の**後**に配置（目次の下に表示される）
2. `<div>`ラッパーは使わない
3. 見出しは`<p><strong>`で表現（見出しタグは使わない）
4. スタイルは`<ul>`に直接適用

## 4. B2B LP設計思想

### 3.1 「判断LP」という考え方

LPは「CVを最大化する装置」ではない。

**LPの役割**:
- 企業が採用すべきかどうかを判断できる状態にする
- 判断が終わった人が、迷わず問い合わせできる状態にする

**やらないこと**:
- 問い合わせを増やすための煽り
- 「今すぐ」「簡単」などの文言追加
- バナー広告的なCTA配置

### 3.2 LP構成

| LP種別 | URL | 役割 |
|--------|-----|------|
| 総論LP | /blog/b2b-recruitment-consulting/ | タイ・ベトナム両方の判断材料 |
| ベトナムLP | /blog/vietnam-recruitment-consulting/ | ベトナム固有の論点に特化（Page ID: 8645） |
| タイLP | /blog/thailand-recruitment-consulting/ | タイ固有の論点に特化（Page ID: 8648） |

### 3.3 内部リンク設計（Type A/B/C）

既存記事からLPへのリンクは記事の性格によって分類：

| Type | 対象記事 | リンク要否 |
|------|----------|------------|
| A | 企業人事が判断で止まる記事（採用ガイド、労働法等） | 必須 |
| B | 制度・市場解説系（祝日、赴任準備等） | 条件付き |
| C | 求職者向け・一般情報 | 不要 |

## 5. CTA設計ルール

### 4.1 基本ルール

#### CTA数（記事の長さ別）

| 記事の長さ | CTA数 | 配置位置 |
|-----------|-------|----------|
| 長い（1万字以上） | **3個** | 10-20%地点、50%地点、末尾（関連記事・出典の前） |
| 中規模 | **2個** | 中段、末尾（関連記事・出典の前） |
| 短い | **1個** | 末尾（関連記事・出典の前） |

#### 配置の原則

| 項目 | ルール |
|------|--------|
| 連続配置 | **禁止**（2つのCTAが連続で並ばない） |
| 配置位置 | 判断完了地点の直後、またはセクションの切れ目 |
| 末尾CTA | 必ず**関連記事・出典の前**に配置 |

#### 長い記事（1万字以上）の配置目安

| 位置 | 配置場所 |
|------|----------|
| 1つ目（10-20%） | 導入・概要セクション終了後の切りのいいところ |
| 2つ目（50%） | 本文中盤、主要セクション終了後の切りのいいところ |
| 3つ目（末尾） | 関連記事・出典の直前 |

### 4.2 CTAテンプレート

**Light（補助記事向け）**: 押し売り感を避け、相談"可能"を伝える
**Standard（信頼構築記事向け）**: 相談を"推奨"するレベル
**Strong（問い合わせ直前記事向け）**: 明確に相談を促す

詳細は `CTA_テンプレート.md` を参照（9パターン）。

以下のパターンは `cta-box` クラスを持たないが、**CTAとしてカウントする**。CTA数の上限（最大2個）を適用する際に含めること。

**識別方法**: 「東南アジア進出・人材採用のご相談」という見出し + お問い合わせリンク

#### パターン2: 専門家にご相談ください

```html
<div style="...">
<h3>専門家にご相談ください</h3>
<p>キャリアリンクアジアでは、東南アジアでのビジネス展開をサポートしています。人材採用についてお気軽にお問い合わせください。</p>
```

**識別方法**: 「専門家にご相談ください」という見出し + お問い合わせリンク

#### 共通識別ルール

以下の要素が含まれるブロックはCTAとして認識する：
- キャリアリンクアジアへの相談・問い合わせを促す見出し
- 「お問い合わせはこちら」等のリンク
- 会社紹介・サービス説明のテキスト

## 6. 記事品質基準

### 5.1 情報源の優先順位

| 優先度 | 情報源 |
|-------|-------|
| 1位 | 一次情報（政府・公的機関、法律原文） |
| 2位 | 現地語情報（タイ語・ベトナム語） |
| 3位 | 日本語情報（JETRO、大使館等）※参考程度 |

**日本語情報だけを参照して記事を作成することは禁止。**

### 5.2 品質チェック5原則

1. **正確性**: 情報が正しく、検証可能
2. **鮮度**: 情報が最新
3. **網羅性**: 読者の疑問をすべてカバー
4. **明瞭性**: 競合より分かりやすい
5. **実用性**: 読者が行動できる

### 5.3 記事構成

1. 導入文
2. 本記事のポイント（要約）
3. 本文（h2/h3で構造化）
4. FAQ
5. まとめ
6. CTA
7. 関連記事（内部リンク）

## 10. 問い合わせ対応の設計思想

### 役割の定義

**私たちの役割は、売ることではない。企業が正しく判断できる状態を作ること。**

### 問い合わせ分類（A/B/C）

| 分類 | 特徴 | 対応 |
|------|------|------|
| A | すぐ依頼したい | 高温度 |
| B | 条件を整理したい | 中温度（基本ここ） |
| C | 情報収集だけ | 低温度 |

**迷ったらBに分類する。**

### 必須ヒアリング5項目

1. 職種
2. 勤務地
3. 給与レンジ
4. 日本語要件
5. 希望時期

### 判断結果

| 判断 | 意味 | アクション |
|------|------|------------|
| ◎ | 採用可能 | 正式依頼へ |
| △ | 条件調整必要 | 調整案を提示 |
| × | 難しい | 理由を伝えて終了 |

**「検討します」で引き延ばすのは禁止。必ず判断を出す。**
