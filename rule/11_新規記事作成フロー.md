# 新規記事作成フロー

## 作成日: 2026-01-20

---

## フロー概要

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│ドラフト作成 │ ──→ │ 品質チェック │ ──→ │  合格判定   │
└─────────────┘     └─────────────┘     └──────┬──────┘
                                               │
                         ┌─────────────────────┼─────────────────────┐
                         │                     │                     │
                         ▼                     ▼                     │
                   ┌──────────┐          ┌──────────┐              │
                   │  不合格  │          │   合格   │              │
                   └────┬─────┘          └────┬─────┘              │
                        │                     │                     │
                        ▼                     ▼                     │
                   ┌──────────┐          ┌──────────┐              │
                   │ リライト │ ─────→  │次の記事へ│              │
                   └──────────┘          └──────────┘              │
                        │                                          │
                        └──────────────────────────────────────────┘
                              （リライト後、再度品質チェック）
```

---

## STEP 1: ドラフト作成

### 1.1 事前確認

- [ ] 既存記事一覧（計画/新規記事作成リスト_2026-01.md）で重複確認
- [ ] ターゲットキーワードの確認
- [ ] 1記事1テーマ（1キーワード）を厳守

### 1.2 記事作成

| 項目 | 要件 |
|------|------|
| 文字数 | 競合以上（目安20,000字以上、テーマによる） |
| 構成 | 目次、本文（複数セクション）、FAQ、CTA |
| フォーマット | HTML形式、`<article class="job-guide">` |
| 保存先 | `draft/` フォルダ |
| ファイル名 | `{記事ID}_{タイトル概要}.html` 例: `TH-15_タイ解雇規定完全解説.html` |

### 1.3 必須要素

- [ ] 導入文（記事の概要と読者メリット）
- [ ] 目次（`<nav class="toc">`）
- [ ] 本文セクション（`<section id="xxx">`）
- [ ] 表・リスト（情報を視覚的に整理）
- [ ] FAQ（5〜10問）
- [ ] まとめセクション
- [ ] CTAボックス（`<div class="cta-box">`）

---

## STEP 2: 品質チェック

### 2.1 チェック項目一覧

#### A. 情報の正確性・信頼性（必須）

| # | チェック項目 | 合格基準 |
|---|------------|---------|
| A1 | 一次情報源の使用 | 政府・公的機関の情報を根拠にしている |
| A2 | 法令・数字の正確性 | 法律名、条文番号、税率、金額等が正確 |
| A3 | 最新性 | 2025-2026年のデータを使用 |
| A4 | 事実と意見の区別 | 推測・意見は明確に区別されている |

#### B. 競合優位性（必須）

| # | チェック項目 | 合格基準 |
|---|------------|---------|
| B1 | 情報量 | 競合上位10記事より網羅的 |
| B2 | 具体性 | 具体的な数字・手順・条文番号がある |
| B3 | 独自価値 | 競合にない情報・視点がある |
| B4 | 読みやすさ | 表・リストで情報が整理されている |

#### C. 技術要件（必須）

| # | チェック項目 | 合格基準 |
|---|------------|---------|
| C1 | CTA設定 | 求職者向けCTA（/jobseeker/register）が正しく設定 |
| C2 | HTML構造 | 見出しの階層が正しい（h2→h3→h4） |
| C3 | 内部リンク | 関連する既存記事へのリンクがある |
| C4 | 禁止ワード | 「法人設立」を使用していない |

### 2.2 チェック実施方法

```
【チェック手順】
1. 作成した記事を通読
2. 上記チェック項目A1〜C4を順番に確認
3. 可能であれば競合上位記事と比較（WebSearch使用）
4. 合格/不合格を判定
```

### 2.3 判定基準

| 判定 | 条件 |
|------|------|
| **合格** | A1〜A4、B1〜B4、C1〜C4すべて満たす |
| **不合格** | いずれか1つでも満たさない |

---

## STEP 3: 合格/不合格後の対応

### 3.1 合格の場合

1. Todoリストで該当記事を「completed」に更新
2. 次の記事のドラフト作成に進む

### 3.2 不合格の場合

1. 不合格理由を特定（どのチェック項目に該当するか）
2. 該当箇所をリライト
3. 再度品質チェックを実施
4. 合格するまで繰り返す

---

## 品質チェック結果の記録

各記事のチェック結果は、以下の形式で記録する（任意だが推奨）：

```
## 記事ID: TH-15
## タイトル: タイの解雇規定完全解説
## チェック日: 2026-01-20

### チェック結果
| 項目 | 結果 | 備考 |
|------|------|------|
| A1 一次情報 | ✅ | 労働者保護法の条文引用あり |
| A2 正確性 | ✅ | 第118条、第119条の条文番号記載 |
| A3 最新性 | ✅ | 2019年改正内容を反映 |
| A4 事実/意見 | ✅ | - |
| B1 情報量 | ✅ | 約25,000字 |
| B2 具体性 | ✅ | 退職金計算表、第119条6事由を網羅 |
| B3 独自価値 | ✅ | 日系企業向け実務アドバイスあり |
| B4 読みやすさ | ✅ | 表12個、リスト多数 |
| C1 CTA | ✅ | /register リンク設定済み |
| C2 HTML | ✅ | h2→h3→h4の階層正しい |
| C3 内部リンク | ⚠️ | 未設定（公開時に追加予定） |
| C4 禁止ワード | ✅ | 使用なし |

### 判定: 合格（C3は公開時に対応）
```
