# 記事品質マニュアル

**このマニュアルは、すべての記事作成・リライト作業において必ず遵守すること。**

---

## 最重要原則

### 目標: 東南アジア情報で日本語No.1になる

1. **転職情報**: 東南アジアで働きたい日本人にとって、最も信頼され、最も役立つ情報を提供する
2. **生活情報**: 東南アジアに住む日本人にとって、最高品質の生活ガイドとなる

**すべての記事は、この目標を達成するために書かれなければならない。**

---

## 国別・コンテンツ優先順位（必須）

### 国別優先順位

| ランク | 国 | 説明 |
|--------|-----|------|
| **SS** | **ベトナム、タイ** | 最優先。日本人にとって必須サイトを目指す |
| **S** | インドネシア | 高優先 |
| **A** | シンガポール、ミャンマー、フィリピン、マレーシア | 中優先 |
| **B** | カンボジア、ラオス | 低優先 |

**ベトナムとタイの記事を最優先で充実させること。**

### 作業優先順位の原則（必須）

**記事の作成・改善作業は、常にSS優先（ベトナム・タイ）で進めること。**

| 優先度 | 作業内容 |
|--------|---------|
| **最優先** | SS国（ベトナム・タイ）の就職・転職記事 |
| **2位** | SS国（ベトナム・タイ）の政治・社会・経済記事 |
| **3位** | SS国（ベトナム・タイ）の生活記事 |
| **4位** | SS国（ベトナム・タイ）の観光記事 |
| **その後** | S〜B国の記事（同様にカテゴリ優先順位に従う） |

**SS国の記事が完了するまで、他の国の記事改善に着手しない。**

### 作業開始前の確認フロー（必須）

**新しい記事改善・作成を始める前に、必ず以下の順序で確認すること：**

1. **SS国の就職・転職記事**：未完了のものがあれば、それを優先
2. **SS国の政治・社会・経済記事**：1が完了したら、これに着手
3. **SS国の生活記事**：2が完了したら、これに着手
4. **SS国の観光記事**：3が完了したら、これに着手
5. **S国以下**：SS国の全カテゴリが完了したら、次の国へ

**この順序を飛ばしてはいけない。生活記事を作成する前に、政治・社会・経済記事の状況を確認すること。**

### 新規記事作成のワークフロー（必須）

**新しいカテゴリの記事作成を始める際は、必ず以下の手順を踏むこと：**

#### Step 1: キーワード調査
```
1. Web検索で主要キーワードを調査
2. 競合サイトの記事タイトル・内容を確認
3. 検索ボリュームが高いキーワードを特定
4. 日本語で情報が少ない分野を特定（差別化ポイント）
```

#### Step 2: 記事計画の作成
```
1. 調査結果を「改善進捗.md」に反映
2. 各記事のターゲットキーワードを明記
3. 優先順位を決定（SS国優先）
4. 作成順序を決定
```

#### Step 3: 実行
```
1. 計画に基づき、優先度の高い記事から作成
2. 1記事完成ごとに進捗ファイルを更新
3. ユーザー確認を依頼
4. 次の記事へ
```

**「調査なしでいきなり記事作成」は禁止。必ず計画を立ててから実行すること。**

### URLスラッグのルール（必須）

**記事のURLは簡潔で分かりやすいものにすること。**

#### 基本ルール

| キーワードを含む | `thailand-living-cost` | `article-7676` |

| 小文字のみ | `bangkok-hospital` | `Bangkok-Hospital` |

#### スラッグの命名規則

```
[国名]-[トピック]-[年（必要な場合）]

例:
- vietnam-minimum-wage-2026
- thailand-labor-law-2025
- vietnam-living-cost
- bangkok-job-market
- thailand-medical-guide
```

**日本語タイトルをそのままURLにしない。必ず英語の簡潔なスラッグを設定すること。**

### 政治・社会・経済カテゴリのガイドライン（重要）

**このカテゴリは日本語記事が少なく、当サイトの強みになる分野。ただし慎重な執筆が必要。**

#### 基本方針

| 項目 | ガイドライン |
|------|-------------|
| **情報源** | 現地語（ベトナム語/タイ語）の一次情報を必ず参照 |
| **スタンス** | 事実の伝達に徹する。批判的・評価的な記述は避ける |
| **更新頻度** | 経済データ、法改正は定期的に更新が必要 |
| **価値** | 日本語で得られない情報を提供することが差別化ポイント |

#### 政治関連記事の注意事項

**政治はセンシティブな分野。以下のルールを厳守すること。**

| やるべきこと | やってはいけないこと |
|-------------|---------------------|
| 事実を客観的に伝える | 政府・政策への批判 |
| 公式発表・公式統計を引用 | 個人的な政治的見解の表明 |
| ビジネスに関連する情報に焦点 | 政治的に対立する内容 |
| 「〜と発表された」等の客観表現 | 「〜は問題だ」等の評価表現 |

### 絶対禁止事項：政府・政策への批判

**政府・政策への批判は絶対にしてはいけません。**

当サイトは日本人がベトナム・タイで働き、生活するための情報を提供するサイトです。政治的な批判は：

1. **現地での事業継続リスク** - 政府批判は現地当局との関係悪化を招く
2. **読者への不利益** - 批判的な記事を読んだ読者が誤解する可能性
3. **サイトの信頼性低下** - 中立性を失うことで情報源としての価値が下がる

**以下は絶対に書いてはいけない：**
- 「〇〇政府の政策は問題がある」
- 「〇〇の対応は不十分だ」
- 「〇〇政権は〜すべきだ」
- 「〇〇の制度は改善が必要」

**代わりに使うべき表現：**
- 「〇〇政府は〜と発表した」（事実の伝達）
- 「〇〇の制度は〜となっている」（制度の説明）
- 「〇〇によると、〜とのことです」（情報源の明示）

**政治・政策に関しては、事実の伝達のみに徹すること。評価・批判・意見は一切含めない。**

## 記事構成の原則（重要）

### 1記事1テーマの原則

**1つの記事には1つのテーマのみを扱うこと。複数のテーマを1記事にまとめない。**

#### なぜ1記事1テーマか

| 理由 | 説明 |
|------|------|
| **SEO効果** | 1つのキーワードに集中することで検索順位が上がりやすい |
| **読者の理解** | テーマが明確で、読者が目的の情報を見つけやすい |
| **記事の深さ** | 1つのテーマを深掘りでき、網羅性が高まる |
| **更新のしやすさ** | 情報が変わったとき、該当記事のみ更新すれば済む |

#### 分割すべきテーマの例

| ❌ 1記事にまとめるべきでない | ✅ 分割後の記事 |
|---------------------------|----------------|
| タイ最低賃金・労働法ガイド | ① タイ最低賃金ガイド<br>② タイ外国人雇用規制ガイド（日本人求職者向け）<br>③ タイ労働法ガイド（タイ人を雇う経営者向け） |
| ベトナム生活費・住居ガイド | ① ベトナム生活費ガイド<br>② ベトナム住居・賃貸ガイド |
| シンガポール転職・ビザガイド | ① シンガポール転職ガイド<br>② シンガポールビザ・労働許可証ガイド |

#### 1記事にまとめてよいテーマの例

密接に関連するテーマは1記事でOK：

| 1記事でOK | 理由 |
|----------|------|
| 最低賃金の金額＋地域別比較 | 同一テーマの詳細 |
| 生活費の内訳＋都市別比較 | 同一テーマの詳細 |
| 転職活動の流れ＋面接対策 | 転職プロセスとして一体 |

### ターゲット読者による分割（重要）

**同じ「労働法」でも、ターゲット読者が異なる場合は必ず分割する。**

#### 労働法関連記事の分割ルール

| ターゲット読者 | 記事テーマ | 内容 |
|--------------|-----------|------|
| **外国人として働く日本人** | 外国人雇用規制ガイド | ビザ、ワークパーミット、外国人最低給与、BOI規制など |
| **経営者・管理者としてタイ人を雇う日本人** | 労働法ガイド（現地人雇用編） | 産休、解雇規制、労働時間、残業代、社会保険など |

#### なぜ分割すべきか

| 観点 | 外国人として働く日本人 | タイ人を雇う経営者 |
|------|----------------------|-------------------|
| **知りたいこと** | 自分のビザ・給与規制 | タイ人従業員の権利・義務 |
| **立場** | 被雇用者 | 雇用者 |
| **法律の関係** | 自分に適用される規制 | 従業員に適用すべき規制 |

**「自分が雇われる側」と「人を雇う側」は全く違う視点。1記事にまとめると読者が混乱する。**

#### 分割の具体例

| ❌ 1記事にまとめるべきでない | ✅ 分割後の記事 |
|---------------------------|----------------|
| タイ労働法・雇用規制ガイド | ① タイ外国人雇用規制ガイド（日本人求職者向け）<br>② タイ労働法ガイド（タイ人を雇う経営者向け） |
| ベトナム労働法完全ガイド | ① ベトナム外国人労働許可証ガイド（日本人求職者向け）<br>② ベトナム労働法ガイド（現地人を雇う経営者向け） |

#### 判断基準

「この情報を探している人は、もう一方の情報も必ず必要か？」

- **はい** → 1記事でOK
- **いいえ** → 分割すべき

**例:** 最低賃金を調べている人 ≠ 労働法の詳細を知りたい人
→ 分割すべき

#### 分割した場合の内部リンク（必須）

**テーマを分割した場合は、必ず相互に内部リンクを貼ること。**

| 記事 | 追加すべきリンク |
|------|-----------------|
| タイ最低賃金ガイド | 「詳しい労働法はこちら → タイ労働法ガイド」 |
| タイ労働法ガイド | 「最低賃金の詳細はこちら → タイ最低賃金ガイド」 |

内部リンクの効果：
- **読者の利便性向上**：関連情報にすぐアクセスできる
- **SEO効果**：サイト内の回遊性が上がり、検索順位向上
- **直帰率低下**：複数記事を読んでもらえる

## 品質基準（絶対条件）

### 記事が満たすべき5つの条件

| # | 条件 | 説明 |
|---|------|------|
| 1 | **正確性** | 情報は正確で、最新で、検証可能であること |
| 2 | **網羅性** | 読者が知りたいことをすべてカバーしていること |
| 3 | **具体性** | 抽象的な説明ではなく、具体的な情報を提供すること |
| 4 | **独自性** | 他サイトにはない価値を提供すること |
| 5 | **実用性** | 読者が実際に行動できる情報を提供すること |

### 品質の低い記事の特徴（これを避ける）

- 汎用的な説明文が多い（「〜が重要です」「〜を検討しましょう」）
- 具体的な数字、名前、住所、料金がない
- どのサイトにも書いてある情報しかない
- 読んでも何をすればいいかわからない
- 情報が古い、または検証されていない

## 競合分析（必須プロセス）

### なぜ競合分析が必要か

「日本語で東南アジアNo.1」になるためには、競合他社の記事を上回る必要がある。

### 競合分析の手順

#### Step 1: 検索キーワードの特定

記事のターゲットキーワードを明確にする。

```
例: 「タイ 転職 エージェント」「バンコク 日本食レストラン」「ベトナム ビザ 取得方法」
```

#### Step 2: 競合ページの調査

Google検索で上位5〜10位のページを確認し、以下を分析する：

| 分析項目 | 確認内容 |
|---------|---------|
| 情報量 | 文字数、見出し数、紹介している項目数 |
| 具体性 | 具体的な数字、名前、住所、料金の有無 |
| 網羅性 | カバーしているトピックの範囲 |
| 更新日 | 情報の新しさ |
| 独自情報 | 他にはない情報の有無 |

#### Step 3: 競合分析シートの作成

```markdown
## 競合分析シート

### ターゲットキーワード: [キーワード]

### 競合ページ一覧

| 順位 | サイト名 | URL | 文字数 | 紹介数 | 強み | 弱み |
|-----|---------|-----|-------|-------|------|------|
| 1 | | | | | | |
| 2 | | | | | | |
| 3 | | | | | | |

### 競合に勝つために必要なこと
1.
2.
3.

### 当サイト記事の差別化ポイント
1.
2.
3.
```

#### Step 4: 競合を上回る計画

競合分析に基づき、以下を明確にする：

- **量で勝つ**: 競合より多くの情報を載せる
- **質で勝つ**: 競合より正確・詳細な情報を載せる
- **独自性で勝つ**: 競合にない情報を載せる

**競合を上回る明確な計画なしに記事を公開してはいけない。**

## リライト品質比較（最重要）

### 絶対ルール

**リライト後の記事が、リライト前の記事より品質が高い場合のみ更新する。**

品質が同等または低下した場合、更新しない。

### 品質比較の手順

#### Step 1: 定量比較

| 項目 | 元記事 | リライト後 | 判定 |
|------|-------|----------|------|
| 文字数 | | | 元記事 ≦ リライト後 → OK |
| 見出し数 | | | 元記事 ≦ リライト後 → OK |
| 紹介企業/サービス数 | | | 元記事 ≦ リライト後 → OK |
| 具体的情報数（住所、電話番号、料金等） | | | 元記事 ≦ リライト後 → OK |
| 内部リンク数 | | | 元記事 ≦ リライト後 → OK |
| 外部リンク数 | | | 元記事 ≦ リライト後 → OK |

**1項目でも「元記事 > リライト後」があれば、原則として更新しない。**

#### Step 2: 定性比較

以下の観点で、元記事とリライト後を比較する：

| 観点 | 質問 | 判定基準 |
|------|------|---------|
| 具体性 | 汎用的な説明が増えていないか？ | 具体的情報が減っていれば NG |
| 独自性 | 元記事の独自情報が消えていないか？ | 独自情報が消えていれば NG |
| 実用性 | 読者が行動しやすくなったか？ | 行動しにくくなっていれば NG |
| 読みやすさ | 構成・文章は改善されたか？ | 悪化していれば NG |
| 正確性 | 情報は正確か？ | 不正確な情報があれば NG |

#### Step 3: 総合判定

```markdown
## リライト品質比較レポート

### 対象記事

- URL:
- タイトル:

### 定量比較結果
| 項目 | 元記事 | リライト後 | 判定 |
|------|-------|----------|------|
| 文字数 | | | |
| 見出し数 | | | |
| 紹介数 | | | |
| 具体的情報数 | | | |

### 定性比較結果
| 観点 | 判定 | 理由 |
|------|------|------|
| 具体性 | OK/NG | |
| 独自性 | OK/NG | |
| 実用性 | OK/NG | |
| 読みやすさ | OK/NG | |
| 正確性 | OK/NG | |

### 総合判定
- [ ] すべてOK → 更新する
- [ ] 1つでもNG → 更新しない、修正する

### 判定理由
（なぜ更新する/しないのかを記載）
```

## 品質低下を防ぐための禁止事項

### 絶対にやってはいけないこと

1. **元記事を読まずにリライトする**
2. **競合分析せずに記事を書く**
3. **具体的情報を汎用的な説明に置き換える**
4. **情報を検証せずに記載する**
5. **品質比較せずに更新する**
6. **「新しい=良い」と思い込む**
7. **量を増やすことだけを目的にする**

### 特に注意すべきパターン

| パターン | なぜ危険か | 対策 |
|---------|----------|------|
| 「〜が重要です」で終わる文 | 具体性がない | 具体的な方法・例を追加 |
| 「〜を検討しましょう」 | 行動に繋がらない | 具体的な手順を記載 |
| 「さまざまな〜があります」 | 情報量ゼロ | 具体例を列挙 |
| 一般論の羅列 | 独自性がない | 独自の視点・情報を追加 |

## 品質チェックの2つの絶対条件（最重要）

### 常に意識すべき2つの質問

すべての記事作成・リライト時に、以下の2つの質問に「はい」と答えられなければ公開しない。

| # | 質問 | 確認方法 |
|---|------|---------|
| **1** | **他の記事より高品質か？** | 競合分析を実施し、情報量・具体性・独自性で上回っているか確認 |
| **2** | **過去の自分の記事より高品質か？** | 既存記事と比較し、文字数・構造・情報量が向上しているか確認 |

### なぜこの2つが重要か

1. **競合より高品質** → 検索で1位を取れる
2. **過去より高品質** → サイト全体の品質が継続的に向上する

**この2つの条件を満たさない記事は、公開する価値がない。**

## 情報の正確性確保

### 一次情報の参照（重要）

**東南アジアの情報を扱う際は、現地語の一次情報を必ず参照すること。**

| 国 | 参照すべき言語 | 一次情報源の例 |
|----|--------------|---------------|
| ベトナム | **ベトナム語** | 政府公式サイト、現地メディア、労働法原文 |
| タイ | **タイ語** | BOI（投資委員会）、労働省、現地求人サイト |
| インドネシア | **インドネシア語** | 移民局、労働省、BKPM |
| シンガポール | 英語 | MOM、ICA、政府公式サイト |
| マレーシア | 英語/マレー語 | Immigration Department、MITI |

**日本語の二次情報だけに頼ってはいけない。** 日本語サイトの情報は古い、または不正確なことが多い。

### 情報源の優先順位

1. **現地語の公式サイト・公式発表**（最優先）
2. **英語の公式サイト**
3. **日本語の公式サイト（大使館等）**
4. **信頼できる現地報道機関**
5. **業界団体・専門機関**
6. **実体験・取材情報**

### 出典の明記（必須）

**一次情報（政府関係、公式見解など）を参照した場合は、必ず出典を明記すること。**

#### 出典として使用できる情報源（厳守）

<div style="background: #fff3e0; border: 3px solid #e65100; border-radius: 8px; padding: 20px; margin: 20px 0;">

**出典に使用できるのは、一次情報のみ。コンサルティング会社や二次情報サイトは出典にしてはいけない。**

| 使用OK（一次情報） | 使用NG（二次情報） |
|-------------------|-------------------|
| 政府機関（労働省、移民局、BOI等） | コンサルティング会社（東京コンサルティング、Deloitte等） |
| 現地語の公式サイト・法律原文 | 日本語の解説サイト |
| JETRO、外務省、大使館 | 法律事務所のコラム |
| 現地の公式統計局 | ニュースサイトの解説記事 |
| 社会保険機関（VSS等） | ブログ・まとめサイト |

**なぜコンサル会社を出典にしてはいけないか：**

1. **一次情報ではない** - コンサル会社も政府情報を翻訳・解釈しているだけ
2. **解釈の誤りがある可能性** - 翻訳や解釈の過程で情報が変質することがある
3. **更新されない可能性** - 法改正後も古い記事が残っていることが多い
4. **信頼性の担保ができない** - 政府公式と異なる場合、責任を問えない

**代わりにすべきこと：**

1. コンサル会社の記事は「調査のきっかけ」として使用する
2. 必ず一次情報（政府サイト、法律原文）で内容を確認する
3. 出典には一次情報のURL・名称のみを記載する

</div>

#### 出典の記載方法

記事末尾に「情報の出典」セクションを設け、以下の形式で記載する：

```html
<p style="font-size: 12px; color: #666; margin-top: 40px; border-top: 1px solid #ddd; padding-top: 16px;">
<strong>情報の出典:</strong><br>
・タイ労働保護法（2025年改正）: <a href="https://...">Tilleke & Gibbins</a><br>
・ベトナム労働法: <a href="https://...">Vietnam.vn</a><br>
・シンガポールMOM: <a href="https://www.mom.gov.sg/...">Ministry of Manpower</a><br>
<br>
※記事の情報は20XX年X月時点のものです。最新情報は各国の公式サイトをご確認ください。
</p>
```

#### 出典が必要な情報

| 情報の種類 | 出典が必要か | 例 |
|-----------|-------------|-----|
| 法律・制度 | **必須** | 産休日数、ビザ要件、労働法 |
| 公式データ | **必須** | 政府統計、最低賃金、為替レート |
| 一般的事実 | 不要 | 「バンコクは首都」など |
| 体験談 | 出典または「取材より」 | インタビュー内容 |

#### リンクの貼り方ルール

**一次情報（法律、政府統計等）のみリンクを貼る。それ以外はリンク不要。**

| 情報の種類 | リンク | 例 |
|-----------|--------|-----|
| 一次情報（政府公式、法律、公式統計） | **必須** | 労働省、移民局、JETRO等 |
| 二次情報・一般論・経験則 | **不要** | 「多くの人が〜」「一般的に〜」 |

**理由:**
- 一次情報へのリンクはSEO効果あり（権威性の証明）
- 二次情報や曖昧な情報にリンクを貼ると信頼性が下がる
- 架空のリンクは絶対禁止

#### 信頼性の高い出典例（政府機関・公的機関のみ）

| 国 | 一次情報源 |
|----|-----------|
| タイ | BOI（投資委員会）、Department of Labour（労働局）、Social Security Office（社会保険事務所）、Royal Thai Government Gazette（官報） |
| ベトナム | MOLISA（労働省）、Vietnam Social Security（VSS）、VnExpress（現地メディア）、政令・通達の原文 |
| シンガポール | MOM（労働省）、MSF（社会家庭省）、ICA（移民局） |
| マレーシア | MOHR（人的資源省）、Immigration Department（移民局） |
| 日本 | JETRO、厚生労働省、外務省 |

**注意**: 法律事務所、コンサルティング会社、会計事務所のサイトは出典として使用不可

以下の情報は必ず公式ソースで確認すること：

- 料金・価格
- 営業時間・定休日
- 住所・連絡先
- ビザ・法律関連
- 手続き・申請方法

## 多角的検証の原則（重要）

### 基本姿勢

**「日本人読者にとって本当に使える情報か？」を多角的に検証すること。**

現地の制度を紹介する際は、以下の点を必ず確認する：

1. **その制度は日本人にも適用されるか？**
2. **駐在員と現地採用で違いはあるか？**
3. **日本の制度との併用・選択は可能か？**

### 駐在員と現地採用の違いを明記する

海外で働く日本人には「駐在員」と「現地採用」の2種類がある。制度の適用が異なる場合は必ず明記すること。

| 項目 | 駐在員 | 現地採用 |
|------|--------|----------|
| 雇用契約 | 日本本社と契約 | 現地法人と契約 |
| 社会保険 | 日本の社会保険継続 | 現地の社会保険 |
| 産休・育休 | 日本の制度を利用可能 | 現地の制度を利用 |
| 給与体系 | 日本本社基準 | 現地法人基準 |

### 制度紹介時の必須確認事項

| 確認項目 | 確認内容 |
|---------|---------|
| 適用対象 | 外国人（日本人）も対象か？ |
| 雇用形態 | 正社員のみか、契約社員も含むか？ |
| 勤続要件 | 何ヶ月以上の勤務が必要か？ |
| 届出・手続き | どのような手続きが必要か？ |
| 日本の制度との関係 | 併用可能か、どちらか一方か？ |

### 記載例

**悪い例（検証不足）:**
```
タイの産休は120日で日本より長いので安心です。
```

**良い例（多角的に検証）:**
```
タイの労働保護法は国籍を問わず適用されるため、現地採用の日本人女性も
120日の産休を取得できます。一方、駐在員として日本の社会保険に
加入している場合は、日本の産休制度（産前6週間+産後8週間）を利用することも
可能です。雇用形態によって利用できる制度が異なるため、
転職前に確認することをおすすめします。
```

### チェックリスト

制度や法律を紹介する際は、以下を確認：

- [ ] 外国人（日本人）にも適用されることを確認したか？
- [ ] 駐在員と現地採用で違いがある場合、明記したか？
- [ ] 日本の制度との関係（併用可否など）を調べたか？
- [ ] 適用条件（勤続期間、届出方法など）を記載したか？
- [ ] 情報源を明記したか？

### 記載してはいけない情報

- 検証できない情報
- 古い情報（更新日が不明な情報）
- 推測・憶測
- 他サイトからの無断転載

### 捏造の禁止（最重要）

**存在しないデータ、調査、統計を作り出してはいけない。**

#### 絶対に禁止されていること

| 禁止事項 | 具体例 |
|---------|--------|
| **架空の調査データ** | 「n=500」「2025年実施」など具体的な数字を捏造 |
| **架空のアンケート結果** | 「CareerLink Asia調べ」など自社調査を偽造 |
| **架空の統計** | 「〇〇%の人が」など裏付けのない数字 |
| **架空の出典** | 実在しない機関・レポートを引用 |

#### 許容される表現

| NG（捏造） | OK（一般論として表現） |
|-----------|----------------------|
| 「調査によると75%が満足」 | 「多くの人が満足している傾向があります」 |
| 「n=500のアンケートでは」 | 「一般的な傾向として」 |
| 「CareerLink Asia調べ」 | 「当社の経験上」「相談を受ける中で」 |

#### なぜ捏造が禁止されるか

1. **信頼の喪失**: 一度嘘がバレると、サイト全体の信頼が失われる
2. **法的リスク**: 虚偽の情報は景品表示法等に抵触する可能性
3. **ユーザーへの害**: 誤った情報で意思決定させることは害悪

**「それっぽく見せるための数字」を作ることは、絶対にしてはいけない。**

### 記事拡充時の一次情報確認ルール（重要）

<div style="background: #fff3e0; border: 3px solid #e65100; border-radius: 8px; padding: 20px; margin: 20px 0;">

**記事を大幅に拡充する際は、一次情報・現地情報をしっかり調べてから書くこと。**

記事のボリュームを増やす（大幅に拡充する）際に最も注意すべきは「ウソを書かないこと」。文字数を増やすために、検証されていない情報や推測を書き足してはいけない。

#### 記事拡充時の必須プロセス

| ステップ | 内容 | 確認事項 |
|---------|------|---------|
| **1. 情報収集** | 追加したい内容について一次情報を調査 | 政府公式サイト、法律原文、公式統計を確認したか？ |
| **2. 検証** | 収集した情報が正確か複数ソースで確認 | 1つの情報源だけに頼っていないか？ |
| **3. 執筆** | 確認できた情報のみを記載 | 出典を明記したか？ |
| **4. レビュー** | 書いた内容が一次情報と一致するか確認 | 推測や憶測が混ざっていないか？ |

#### 拡充時にやってはいけないこと

| 禁止事項 | 理由 |
|---------|------|
| **文字数稼ぎのための一般論追加** | 価値のない内容が増え、読者の時間を奪う |
| **未確認の詳細情報の追加** | 誤った情報が読者に害を与える |
| **他サイトの情報をそのまま使う** | 二次情報は誤りを含む可能性がある |
| **「〜と思われる」「〜でしょう」の多用** | 憶測は記事の信頼性を下げる |

#### 拡充時に追加すべき内容

| 追加OK | 例 |
|--------|-----|
| **一次情報で確認できた詳細** | 法律の具体的な条文、公式の手続き手順 |
| **公式統計データ** | 政府発表の数値、公式レポートの引用 |
| **実用的なアドバイス（経験則として表現）** | 「〜ことが多いです」「〜を確認することをおすすめします」 |
| **日本人読者向けの補足情報** | 日本との比較、日本語での手続き方法 |

</div>

**「文字数を増やすこと」ではなく「読者に価値ある情報を増やすこと」が目的。**

### 断定表現と柔らかい表現の使い分け（重要）

**一次情報から確認できる内容は断定OK。それ以外は柔らかい表現を使う。**

#### 基本ルール

| 情報の種類 | 表現 | 出典 |
|-----------|------|------|
| **一次情報**（法律、政府公式、公式統計） | **断定OK** | **必ず明記** |
| **二次情報**（メディア記事、一般サイト） | 柔らかい表現 | 可能なら明記 |
| **経験則・一般論**（「〜が多い」等） | 柔らかい表現 | 不要 |

#### 一次情報の例（断定OK + 出典必須）

```
タイの産休は労働保護法により120日と定められています。
（出典：タイ労働保護法 2025年改正）
```

```
ベトナムで外国人が就労するには労働許可証が必要です。
（出典：ベトナム政令219/2025/ND-CP）
```

#### 二次情報・経験則の例（柔らかい表現）

| 避けるべき表現 | 推奨表現 |
|--------------|---------|
| 〜です | 〜**ことが多いです**、〜**の場合が多いです** |
| 〜になります | 〜**になることが多いです** |
| 〜が必要です | 〜**が必要になることが多いです** |
| 〜で取得できます | 〜**で取得できることが多いです** |
| 会社が負担します | 会社が負担する**ケースが多いですが、確認が必要です** |

#### 具体例

**一次情報あり（断定OK）:**
```
タイの産休は法律上120日です（出典：タイ労働保護法）。
```

**一次情報なし（柔らかく）:**
```
日系企業では、産休中も給与の一部が支給されることが多いですが、
会社によって異なりますので、雇用契約を確認してください。
```

#### 特に柔らかい表現が必要な分野

以下は会社・個人・時期によって大きく異なるため、断定を避ける：

- **会社の福利厚生**: 会社によって異なる
- **費用・料金**: 変動する、地域差がある
- **所要期間**: 混雑状況、担当官によって変わる
- **実際の運用**: 法律と現場の運用が異なることがある

## 日本人読者視点の徹底（重要）

### 基本原則

**すべての記事は日本人読者の視点で書くこと。**

日本人が海外で働く・生活する際に「本当に知りたいこと」「不安に感じること」を常に意識する。

### 日本人視点で書くべき内容

| 観点 | 日本人が気になること | 記載すべき情報 |
|------|---------------------|----------------|
| **比較** | 日本との違いは？ | 日本の制度・文化との比較 |
| **安全性** | 安全に暮らせる？ | 治安、医療、日本人コミュニティ |
| **言語** | 日本語は通じる？ | 日本語の通用度、英語の必要性 |
| **食事** | 日本食は食べられる？ | 日系スーパー、日本食レストラン情報 |
| **住居** | 日本人向け物件は？ | 日本人駐在員が多いエリア |
| **医療** | 日本語が通じる病院は？ | 日本語対応の医療機関 |
| **コスト** | 日本円で換算すると？ | 日本との物価比較、為替考慮 |

### 具体的な書き方

**悪い例（日本人視点がない）:**
```
タイの産休は120日です。
```

**良い例（日本人視点がある）:**
```
タイの産休は120日で、日本の産前6週間＋産後8週間（計98日）より長い。
さらに、雇用主負担の有給期間も60日と手厚く、日本人女性にとって
働きやすい環境が整っています。
```

### チェック項目

記事を書く際は、以下を必ず確認：

- [ ] 日本との比較情報を入れたか？
- [ ] 日本人が不安に感じる点に答えているか？
- [ ] 日本円での金額換算を入れたか？
- [ ] 日本人が利用できるサービス・施設を紹介したか？
- [ ] 日本人の成功事例・体験談を含めたか？

### 専門用語・外国語の日本語化（必須）

**外国語や専門用語は、一般的な日本語表現に置き換えて使用する。**

読者は転職・海外生活の専門家ではない。カタカナ語や専門用語をそのまま使うと、読者が理解できない。

#### 日本語化の原則

| 避けるべき表現 | 使うべき表現 | 備考 |
|--------------|-------------|------|
| アポスティーユ | **公文書の認証** | 正式名称は括弧で補足可 |
| ワークパーミット | **労働許可証** | 日本語を先に書く |
| テンポラリーレジデンスカード | **一時滞在許可証** | カタカナ語は避ける |
| ブリッジSE | **日本とベトナムの橋渡し役となるSE** | 初出時に説明 |
| オフショア開発 | **海外への開発委託** | IT未経験者にも分かる表現 |

#### 書き方のルール

1. **日本語を先に書き、正式名称は括弧で補足**
   - ○: 公文書の認証（アポスティーユ）
   - ×: アポスティーユ（公文書の認証）

2. **説明なしでカタカナ語を使わない**
   - ○: 労働許可証の取得が必要です
   - ×: ワークパーミットの取得が必要です

3. **読者が「これ何？」と思わない文章にする**

**例:**
```html
<!-- 良い例 -->
<div style="background: #e8f5e9; border: 1px solid #4caf50; border-radius: 8px; padding: 15px; margin: 15px 0;">
<h4 style="color: #2e7d32; margin: 0 0 10px 0;">「公文書の認証」とは？</h4>
<p>日本の公的書類（無犯罪証明書、卒業証明書など）を海外で使用する際、
外務省が「この書類は本物です」と証明するスタンプを押す手続きです。
正式名称は「アポスティーユ」といい、無料で取得できます。</p>
</div>
```

## 図解・ビジュアル要素のガイドライン

### 図解の重要性

複雑な情報は文章だけでなく、図解で視覚的に伝えることで読者の理解が深まる。

### HTML/CSS図解の作成ルール

**図解は必ずHTML/CSSで作成し、画像ファイルを使用しない。**

#### なぜHTML/CSSで作成するか

1. **レスポンシブ対応**: 画面サイズに応じて崩れない
2. **編集が容易**: テキストベースなので更新しやすい
3. **SEO対応**: テキスト情報が検索エンジンに読み取られる
4. **読み込み速度**: 画像より軽量

#### 図解のテンプレート例

**1. 比較表（横並び）**
```html
<div style="display: flex; flex-wrap: wrap; gap: 16px; margin: 24px 0;">
  <div style="flex: 1; min-width: 280px; padding: 20px; background: #f8f9fa; border-radius: 8px; border-left: 4px solid #007bff;">
    <h4 style="margin: 0 0 12px; color: #007bff;">タイトル1</h4>
    <p style="margin: 0; color: #333;">説明文...</p>
  </div>
  <div style="flex: 1; min-width: 280px; padding: 20px; background: #f8f9fa; border-radius: 8px; border-left: 4px solid #28a745;">
    <h4 style="margin: 0 0 12px; color: #28a745;">タイトル2</h4>
    <p style="margin: 0; color: #333;">説明文...</p>
  </div>
</div>
```

**2. フロー図（ステップ）— 縦表示が必須**

**重要: フロー図は必ず縦表示にする。横並びは画面幅によって崩れるため禁止。**

```html
<!-- 縦表示のフロー図（推奨） -->
<div style="max-width: 400px; margin: 24px auto;">
  <div style="padding: 15px 20px; background: #1976d2; color: white; border-radius: 8px; text-align: center;">
    <div style="font-size: 0.8em;">STEP 1</div>
    <div style="font-weight: bold;">準備</div>
    <div style="font-size: 0.85em; opacity: 0.9;">詳細説明</div>
  </div>
  <div style="text-align: center; font-size: 1.5em; color: #666; margin: 5px 0;">↓</div>
  <div style="padding: 15px 20px; background: #388e3c; color: white; border-radius: 8px; text-align: center;">
    <div style="font-size: 0.8em;">STEP 2</div>
    <div style="font-weight: bold;">申請</div>
    <div style="font-size: 0.85em; opacity: 0.9;">詳細説明</div>
  </div>
  <div style="text-align: center; font-size: 1.5em; color: #666; margin: 5px 0;">↓</div>
  <div style="padding: 15px 20px; background: #f57c00; color: white; border-radius: 8px; text-align: center;">
    <div style="font-size: 0.8em;">STEP 3</div>
    <div style="font-weight: bold;">完了</div>
    <div style="font-size: 0.85em; opacity: 0.9;">詳細説明</div>
  </div>
</div>
```

**横並びフロー図の禁止理由:**
- スマートフォンで見ると折り返して表示がおかしくなる
- 矢印「→」が途中で切れて意味不明になる
- PCでも画面幅によっては崩れる

**3. 統計・数字の強調**
```html
<div style="display: flex; flex-wrap: wrap; gap: 16px; margin: 24px 0; text-align: center;">
  <div style="flex: 1; min-width: 150px; padding: 24px; background: linear-gradient(135deg, #667eea 0%, #764ba2 100%); border-radius: 12px; color: white;">
    <div style="font-size: 32px; font-weight: bold;">85%</div>
    <div style="font-size: 14px; margin-top: 8px;">女性の満足度</div>
  </div>
  <div style="flex: 1; min-width: 150px; padding: 24px; background: linear-gradient(135deg, #f093fb 0%, #f5576c 100%); border-radius: 12px; color: white;">
    <div style="font-size: 32px; font-weight: bold;">3,000+</div>
    <div style="font-size: 14px; margin-top: 8px;">求人数</div>
  </div>
</div>
```

#### 図解作成時の注意点

| 項目 | ルール |
|------|--------|
| レスポンシブ | `flex-wrap: wrap` と `min-width` を必ず使用 |
| 色の使用 | インラインスタイルで直接指定（外部CSSに依存しない） |
| フォント | システムフォントを使用（特殊フォント不使用） |
| 余白 | `margin` と `padding` を適切に設定 |
| 崩れ防止 | モバイル表示を想定した `min-width` 設定 |

## 進捗管理（必須）

### 改善進捗ファイルの運用

**記事改善の進捗は、専用ファイルで管理すること。**

#### 進捗ファイル: `改善進捗.md`

記事改善を行う際は、必ず以下のファイルを更新する：

```markdown
# 記事改善進捗

## 完了した記事

| 記事ID | タイトル | 改善前文字数 | 改善後文字数 | 完了日 |
|--------|---------|-------------|-------------|--------|
| 7705 | 海外転職で後悔しないために | 4,805 | 27,496 | 2026-01-19 |
| 7738 | インドネシア転職完全ガイド | 6,805 | 28,408 | 2026-01-19 |

## 未改善の記事

| 記事ID | タイトル | 現在の文字数 | 優先度 | 課題 |
|--------|---------|-------------|--------|------|
| 7702 | ベトナムの日系企業で働く | 5,607 | B | 文字数少 |
```

#### 運用ルール

1. **改善完了時**: 「完了した記事」テーブルに追加
2. **作業開始前**: 「未改善の記事」を確認し、重複作業を防ぐ
3. **品質レポート作成時**: 進捗ファイルと照合して最新状態を反映

#### なぜ進捗管理が必要か

- **重複作業の防止**: 既に改善済みの記事に再度時間をかけない
- **進捗の可視化**: どこまで完了したか一目でわかる
- **引き継ぎ対応**: セッションが変わっても作業を継続できる

**進捗ファイルを更新せずに次の作業に進むことは禁止。**
