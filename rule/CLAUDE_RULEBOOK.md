## 1. 最上位目標：東南アジア最高の情報サイトを作る

### なぜ最高のコンテンツが必要か

CareerLink Asiaは人材紹介会社だが、**企業からの問い合わせを得るには、まず「このサイトを見れば間違いない」と言われる情報サイトになる必要がある。**

最高のコンテンツがあるからこそ：
- 企業の人事担当者が信頼して相談する
- 求職者が集まり、人材プールが構築される
- Googleで上位表示され、認知が広がる

### 対象範囲（転職・就職だけではない）

| 分野 | 例 |
|------|-----|
| 転職・就職 | 求人ガイド、人材紹介会社比較、年収相場 |
| 政治・行政 | 政府動向、政策変更、選挙 |
| 経済 | GDP、投資環境、産業動向 |
| 法律 | 労働法、外資規制、ビザ・労働許可 |
| 経理・会計 | 税制、社会保険、給与計算 |
| 社会・文化 | 祝日、マナー、宗教、国民性 |
| 地方 | 工業団地、地方都市、不動産 |
| 生活 | 生活費、住居、医療、教育 |
| 観光 | 都市ガイド、ホテル、移動手段 |

**すべての分野で、日本語で読める最高の情報を提供する。**

### 競合に勝つ唯一の方法

日本語の情報だけを参照していては、絶対に日本一のサイトにはなれない。

- 一次情報（政府・公的機関）を参照する
- 現地語（タイ語・ベトナム語）の情報を参照する
- 競合サイトより網羅的・正確・最新の情報を提供する

## 2. 事業の定義

### 事業の核心

**人材紹介会社**である。東南アジア（特にベトナム・タイ）にある日系企業が人材を探す際に、現地人材および日本人材を紹介する。

### 顧客の優先順位

| 優先度 | 顧客 | 説明 |
|-------|------|------|
| **1位** | **日系企業（採用側）** | ベトナム・タイで人材を探している日系企業 |
| 2位 | 求職者（日本人） | 東南アジアで働きたい日本人 |
| 3位 | 求職者（現地人材） | 日系企業で働きたい現地人材 |

**B2B（企業顧客へのリーチ）が最も重要。**

### 対象地域

**タイ、ベトナム**の2カ国のみに特化。シンガポール、マレーシア、インドネシアは対象外。

## 3. 絶対禁止事項

| # | 禁止事項 | 理由 |
|---|---------|------|
| 1 | 「法人設立」サービスを提案する | 人材紹介会社であり、法人設立サービスは提供していない |
| 2 | 「投資アドバイス」を提供する | 投資コンサルタントではない |
| 3 | 政府・政策を批判する | 現地での事業継続リスク |
| 4 | 架空のデータを捏造する | 「n=500」「CareerLink Asia調べ」など禁止 |
| 5 | 日系企業を批判する | クライアント企業のため |
| 6 | コンサル会社を出典にする | 一次情報のみ使用可 |
| 7 | 手動目次を追加する | WordPressプラグインと重複 |
| 8 | 日本語をURLスラッグにする | SEOに悪影響 |
| 9 | 元記事を読まずにリライトする | 品質低下の原因 |
| 10 | バックアップなしでリライトする | ロールバック不可能 |

## 4. B2B LP設計思想

### 3.1 「判断LP」という考え方

LPは「CVを最大化する装置」ではない。

**LPの役割**:
- 企業が採用すべきかどうかを判断できる状態にする
- 判断が終わった人が、迷わず問い合わせできる状態にする

**やらないこと**:
- 問い合わせを増やすための煽り
- 「今すぐ」「簡単」などの文言追加
- バナー広告的なCTA配置

### 3.2 LP構成

| LP種別 | URL | 役割 |
|--------|-----|------|
| 総論LP | /blog/b2b-recruitment-consulting/ | タイ・ベトナム両方の判断材料 |
| ベトナムLP | /blog/vietnam-recruitment-consulting/ | ベトナム固有の論点に特化 |
| タイLP | （未作成） | タイ固有の論点に特化 |

### 3.3 内部リンク設計（Type A/B/C）

既存記事からLPへのリンクは記事の性格によって分類：

| Type | 対象記事 | リンク要否 |
|------|----------|------------|
| A | 企業人事が判断で止まる記事（採用ガイド、労働法等） | 必須 |
| B | 制度・市場解説系（祝日、赴任準備等） | 条件付き |
| C | 求職者向け・一般情報 | 不要 |

## 5. CTA設計ルール

### 4.1 基本ルール

| 項目 | ルール |
|------|--------|
| CTA数 | **最大2個**（中段+末尾） |
| 連続配置 | 禁止（2つのCTAが連続で並ばない） |
| 配置位置 | 判断完了地点の直後が理想 |

### 4.2 CTAテンプレート

**Light（補助記事向け）**: 押し売り感を避け、相談"可能"を伝える
**Standard（信頼構築記事向け）**: 相談を"推奨"するレベル
**Strong（問い合わせ直前記事向け）**: 明確に相談を促す

詳細は `CTA_テンプレート_B2B.md` を参照。

| ターゲット | URL |
|-----------|-----|
| 求職者向け | https://kyujin.careerlink.asia/jobseeker/register |
| 企業向け | https://kyujin.careerlink.asia/お問い合わせ |

## 6. 記事品質基準

### 5.1 情報源の優先順位

| 優先度 | 情報源 |
|-------|-------|
| 1位 | 一次情報（政府・公的機関、法律原文） |
| 2位 | 現地語情報（タイ語・ベトナム語） |
| 3位 | 日本語情報（JETRO、大使館等）※参考程度 |

**日本語情報だけを参照して記事を作成することは禁止。**

### 5.2 品質チェック5原則

1. **正確性**: 情報が正しく、検証可能
2. **鮮度**: 情報が最新
3. **網羅性**: 読者の疑問をすべてカバー
4. **明瞭性**: 競合より分かりやすい
5. **実用性**: 読者が行動できる

### 5.3 記事構成

1. 導入文
2. 本記事のポイント（要約）
3. 本文（h2/h3で構造化）
4. FAQ
5. まとめ
6. CTA
7. 関連記事（内部リンク）

## 10. 問い合わせ対応の設計思想

### 役割の定義

**私たちの役割は、売ることではない。企業が正しく判断できる状態を作ること。**

### 問い合わせ分類（A/B/C）

| 分類 | 特徴 | 対応 |
|------|------|------|
| A | すぐ依頼したい | 高温度 |
| B | 条件を整理したい | 中温度（基本ここ） |
| C | 情報収集だけ | 低温度 |

**迷ったらBに分類する。**

### 必須ヒアリング5項目

1. 職種
2. 勤務地
3. 給与レンジ
4. 日本語要件
5. 希望時期

### 判断結果

| 判断 | 意味 | アクション |
|------|------|------------|
| ◎ | 採用可能 | 正式依頼へ |
| △ | 条件調整必要 | 調整案を提示 |
| × | 難しい | 理由を伝えて終了 |

**「検討します」で引き延ばすのは禁止。必ず判断を出す。**
